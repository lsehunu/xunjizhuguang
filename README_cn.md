布线规范：所有线路提前定型捆扎，无飞线、无临时改装非标接线，硬件整体集成一体化固定，赛前完成 72 小时不间断拷机稳定性测试，规避赛场接触不良、硬件兼容类故障。


三、项目整体技术路线与完整系统构成概述
3.1 端到端完整技术路线（全流程基于地瓜官方工具链）
整体技术链路：实时图像采集 → 图像预处理模块 → NPU 轻量化模型推理 → 赛事业务逻辑决策 → 可视化结果输出 + 本地日志存储
1.图像采集层：调用地瓜官方 OpenCV 适配接口，读取 USB 摄像头实时视频流，固定画面分辨率、帧率，统一图像输入格式，减少推理前格式转换耗时，适配赛场固定拍摄场景。
2.图像预处理层 
3.基于官方开源图像处理例程，完成画面降噪、尺寸归一化、色域转换、对比度自适应调节，消除赛场光照变化、反光、阴影对识别精度的干扰；预处理代码完全适配 RDK X5 硬件加速指令。
4.NPU 模型推理层：模型训练完成后，使用地瓜官网提供的模型转换工具完成 PT/ONNX 模型量化，转换为适配 RDK X5 NPU 的 Tengine 专用模型文件；推理进程独占 NPU 算力，保障多目标检测毫秒级输出，满足赛事实时性要求。
5.赛事业务逻辑层：自主开发 Python 业务控制程序，对接赛事全部考核规则，包含目标计数、物体定位、状态判别、任务得分统计、异常场景过滤五大核心逻辑；程序预留调试日志接口，赛场可快速查看推理报错、识别漏检问题。
6.结果输出交互层：双输出设计：①屏幕可视化界面，实时叠加目标检测框、类别标签、任务进度；②本地日志自动写入 SD 卡，完整记录每帧识别数据、运行耗时，便于赛后复盘优化。
3.2 系统分层架构完整说明
本系统采用五层分层设计，全部底层驱动、基础组件来源于地瓜机器人官方开源仓库，无第三方非标闭源组件：
1.硬件驱动底层：地瓜官方原厂 BSP 驱动包，统一管理摄像头、显示屏、GPIO 外设、存储设备，解决硬件底层兼容问题，无需自行开发底层驱动，降低赛场调试工作量。
2.嵌入式系统基础层：RDK 官方定制 Ubuntu 系统镜像，预装 NodeHub 开发环境、Tengine 推理引擎、OpenCV 图像处理库、Python 运行环境，系统镜像赛前烧录备份 2 份，防止 SD 卡损坏导致程序丢失。
3.AI 算法推理层：轻量化目标检测模型（YOLO 轻量化版本），经官方量化工具压缩优化，适配 RDK 端侧算力；配套官方推理调度 API，实现多帧并行推理、算力资源动态分配。
4.上层业务应用层：自主编写竞赛任务主程序，串联采集、预处理、推理、判断全流程，针对赛事任务规则定制判断阈值、过滤逻辑，可快速调整参数适配赛场环境变化。
5.可视化交互展示层：基于官方 QT 可视化例程二次开发交互界面，分区展示原始画面、识别结果、系统运行参数、任务完成进度，便于现场裁判直观查看系统运行状态。

五、赛场稳定性保障与前置风险预案
结合 RDK 平台硬件特性与往届赛事常见故障，赛前完成全套稳定性测试与应急方案，纳入备案参考：
1.硬件故障预案 备用物资：备用摄像头、备用 SD 卡（预存完整系统与程序）、备用电源；若出现画面丢失、供电中断，5 分钟内完成硬件替换恢复运行。
2.软件推理卡顿预案 提前预置两套模型权重文件（高精度版 / 极速轻量化版），赛场光照差、延迟过高时一键切换低算力模型，保障画面流畅不丢帧。
3.环境干扰预案 预处理模块内置亮度、对比度自适应调节逻辑，应对赛场强光、背光、反光场景，降低目标漏检、误检概率。
4.程序崩溃预案 主程序增加异常捕获与自动重启机制，后台持续记录运行日志，程序异常后自动重启图像与推理流程，无需人工重新启动开发板。
